## 任务 1：安全修复 — 撤销泄露的 API 密钥（最高优先级）

### 1.1 确认泄露范围

当前 pip 安装包中有 **4 个 `.env` 文件**，泄露了 **5 个不同的敏感凭证**：

| 文件路径 | 泄露的凭证 |
|---------|-----------|
| `.env`（根目录） | `RELAY_API_KEY=eyJhbGciOiJSUzI1NiIs...`（api.trae.ai JWT） |
| `backend/.env` | 同上 JWT + `JWT_SECRET=62894aa2...` |
| `services/.env` | `RELAY_API_KEY=1aecbd9241fc4a05ba6449df281d4a2d.n9fnhFs1O9mukJF4`（智谱 BigModel）+ `DEEPSEEK_API_KEY=sk-438abf7854b0412685c7f64d2bf62276` + `GLM_API_KEY=1aecbd9241fc4a05ba6449df281d4a2d.n9fnhFs1O9mukJF4` |
| `services/backend/.env` | 同 services/.env 的全部密钥 + `JWT_SECRET=5aa44f32cf...` |

**需要轮换的全部密钥**：
1. **api.trae.ai JWT** — `eyJhbGciOiJSUzI1NiIs...`（完整 JWT）
2. **智谱 BigModel API Key** — `1aecbd9241fc4a05ba6449df281d4a2d.n9fnhFs1O9mukJF4`
3. **DeepSeek API Key** — `sk-438abf7854b0412685c7f64d2bf62276`
4. **JWT_SECRET #1** — `62894aa2080a29819e4c0fa9bd904f04c321493407ccd7ac80f0069991d91613`
5. **JWT_SECRET #2** — `5aa44f32cf1a326d44517551034b5f4c20ceb2681496b83712ae77bba1bf1bef`

### 1.2 轮换全部密钥（手动操作，必须在发布前完成）

1. **api.trae.ai**：登录控制台，撤销当前 JWT，生成新 token
2. **智谱 BigModel**：登录 https://open.bigmodel.cn 控制台，撤销 key `1aecbd92...`，生成新 key
3. **DeepSeek**：登录 https://platform.deepseek.com 控制台，撤销 key `sk-438abf...`，生成新 key
4. 生成新的 JWT_SECRET：`openssl rand -hex 32`（只需一个，不要像现在这样有两个不同的）
5. 更新本机 `~/.khyquant/config.json` 或环境变量中的全部密钥

### 1.2b 确认 .env 来源

这 4 个 `.env` 文件可能有两个来源：
- **wheel 打包时带入**（pip_packaging_rules.py 已有排除规则，但可能未生效）
- **源码快照中包含**（`makeSourceSnapshot.js` 可能未排除 `.env`，`khy restore` 解压后暴露）
- **bootstrap 运行时生成**（`services/backend/.env` 标注了 "Generated by khy-os bootstrap"）

必须同时堵住三个入口：
1. `pip_packaging_rules.py` 的排除规则（已有，确认生效）
2. `makeSourceSnapshot.js` 的排除规则（必须添加 `.env` 排除）
3. `.gitignore` 必须包含 `.env`（防止 `git archive` 带入）

### 1.3 确保密钥不进入源码快照

检查 `makeSourceSnapshot.js`（搜索项目中的此文件），确认其排除规则包含：

```bash
grep -n '\.env\|\.env\.local\|INTERNAL_CREDENTIALS\|JWT_SECRET\|API_KEY' $(find . -name "makeSourceSnapshot*" -not -path "*/node_modules/*")
```

如果 `makeSourceSnapshot.js` 没有显式排除 `.env`，必须添加排除逻辑。快照应当：
- **排除**：`.env`、`.env.local`、`*.key`、`*.pem`、`INTERNAL_CREDENTIALS*`
- **包含**：`.env.example`、`.env.template`（如果存在的话，作为模板）

### 1.4 确认 .gitignore 包含敏感文件

```bash
cat .gitignore | grep -E '\.env|credential|secret|key'
```

如果缺失，追加：
```
.env
.env.local
.env.*.local
*.key
*.pem
INTERNAL_CREDENTIALS*
```

### 1.5 创建 .env.example 模板

在项目根目录创建 `.env.example`，包含所有需要的环境变量名但不含真实值：


### 2.2 检查 pip_packaging_rules.py 的排除规则是否完备

```bash
python3 -c "
```

### 2.3 在 AUDIT_FORBIDDEN_FILE_GLOBS 中补充

### 8.1 验证 `publish-from-pip-dual.ps1` 可用

项目已有一份 `scripts/release/publish-from-pip-dual.ps1`，它从 pip 安装的包中导出源码并推送到 git 仓库。确认此脚本：

```bash
# 检查脚本引用的命令是否都存在
grep -E 'khy publish|pip install|git push' scripts/release/publish-from-pip-dual.ps1

# 确认 khy publish origin-code 子命令在安装后可用

```

**注意**：此脚本中硬编码了两个 GitHub 仓库地址：
- `git@github.com:Program-master-leader/KHY-OS.git`（私有）
- `git@github.com:Program-master-leader/khy_os.git`（公开）

确认这两个仓库的 SSH key 已在 Windows 机器上配置，或改为 HTTPS + token 方式。

### 8.2 验证 `khy restore` 和 `khy publish origin-code` 两种还原路径

```bash
# 路径 A：khy restore（直接解压到目录）
khy restore /tmp/test-restore-a --force
ls /tmp/test-restore-a/pyproject.toml
ls /tmp/test-restore-a/services/backend/package.json

# 路径 B：khy publish origin-code（导出为 zip）
khy publish origin-code --out /tmp/test-restore-b
# 解压 zip 验证

ls /tmp/test-restore-b-extracted/*/pyproject.toml
```

两条路径还原的源码树都必须包含完整的开发所需文件。

### 8.3 清理孤儿文件

以下文件不应出现在源码树或 pip 包中，确认从源码快照中排除：

```bash
# scan_test.js — 孤儿测试文件，引用了凭据解析逻辑
rm -f scan_test.js

# .khy_orphan_sweep — 运行时标记文件

```

在 `makeSourceSnapshot.js` 的排除列表中添加：
```
scan_test.js
.khy_orphan_sweep
khy-Trajectory*
```

### 8.4 确认 .npmrc 不泄露

`backend/.npmrc` 包含 `registry=https://registry.npmmirror.com`。这本身不是密钥，但：
- 不应出现在分发包中（强制用户使用特定镜像）
- 应在 `pip_packaging_rules.py` 的 `COPY_EXCLUDE_PATTERNS` 中添加 `.npmrc`
- 或至少在 `makeSourceSnapshot.js` 中排除

### 8.5 确认 DEFAULT_SOURCE_SECRET 安全模型

源码快照的加密密钥 `khy2026` 硬编码在 `sourceSnapshotCrypto.js` 中：
```javascript
const DEFAULT_SOURCE_SECRET = 'khy2026';
```

这意味着任何能读到源码的人（即所有 pip 用户）都能解密快照。这**不是**安全漏洞（源码本就分发给了所有 pip 用户），但需要明确：
- 快照加密是**传输保护**，不是**访问控制**
- 快照中**绝不能**包含任何不应公开的内容（如 `.env`、密钥）
- `RESTORE_WINDOWS.md` 中声明的「`.env` 不入包（安全）」必须名副其实
