1. Spinner 结束后不变为完成态#

现象#

AI 回复完成后，状态行仍显示 ✦ 正在输出... (64s)，未切换为 ✳ 完成 · 64.0s 的灰色终止态。

根因#

StreamingChunker 使用 setImmediate 异步 flush 队列。当 ai().chat() 的 Promise resolve 时：

时序:
  1. chat() resolve
  2. .finally() → turnComplete dispatch → activity.state = 'done'
  3. setImmediate 回调触发 → chunker flush 残留 chunk
  4. CHUNK_RECEIVED dispatch → activity.state 被覆盖回 'streaming'
  5. 后续无 turnComplete，state 永久停在 'streaming'

replStore.js 中 CHUNK_RECEIVED 和 STREAMING_BLOCK_START 两个 action 无条件将 activity.state 设为 'streaming'，覆盖了已到达的 'done' 终态。

修复#

三处协同修复，消除竞争条件：

a) replStore.js — 终态保护

CHUNK_RECEIVED 和 STREAMING_BLOCK_START 不再覆盖 done 状态：

// before
activity: { ...state.activity, state: 'streaming' },

// after
activity: {
  ...state.activity,
  state: state.activity.state === 'done' ? 'done' : 'streaming',
},

b) tuiShell.js — 同步 drain

chat() 返回后、streamingBlockEnd 之前，同步清空 chunker 残留队列：

const result = await ai().chat(text, { onChunk, onStatus });
chunker.drain();   // ← 新增：同步 flush 残留 chunk
// Finalize any open streaming block ...

c) StreamingChunker — 新增 drain() 方法

drain() {
  if (this._queue.length === 0) return;
  const coalesced = this._coalesce();
  for (const item of coalesced) this._dispatch(item);
  this._queue.length = 0;
  this._scheduled = false;
}

d) tuiShell.js — turnComplete 后强制渲染

processInput(trimmed).finally(() => {
  _setBusy(false);
  store.dispatch(actions.turnComplete({}));
  _renderScheduled = false;
  setImmediate(_doRenderAndTrack);  // 绕过 debounce，立即渲染 done 态
});

2. 中文断行（"今天" 被拆到两行）#

现象#

AI 回复中 "...等等。今" 在一行末尾，"天有什么..." 在下一行开头，中间还有空行。"今天" 作为一个语义单元被拆开。

根因#

不是 wrapText 的 bug。AI 模型（Claude）在流式输出时，token 边界恰好落在 "今" 和 "天" 之间，输出中包含了 \n：

实际模型输出: "...等等。今\n天有什么..."

wrapText 按 \n 切分后，"今" 和 "天" 被分到不同的行/段落。

修复#

在 wrapText 入口添加正则预处理，将 CJK 字符间的孤立 \n 合并为连续文本：

// TranscriptViewport.js — wrapText 入口
let cleaned = String(text).replace(
  /(?<=[\u2e80-\u9fff\uf900-\ufaff\ufe30-\ufe6f\uff01-\uff60])\n(?=[\u2e80-\u9fff\uf900-\ufaff\ufe30-\ufe6f\uff01-\uff60])/g,
  ''
);

规则：

  • CJK字\nCJK字 → 合并（今\n天 → 今天）
  • CJK字\n\nCJK字 → 保留段落分隔（。\n\n第 不变）
  • English\nEnglish → 保留（不影响英文换行）

3. Claude Opus 4 显示 200k 而非 1M 上下文#

现象#

欢迎页和 footer 显示 上下文: 200k，但 Claude Opus 4.6 实际支持 1M。

根因#

ai.js 中 MODEL_CAPABILITIES 静态表记录了错误值：

'claude-opus-4': { ..., context: 200000, ... },  // 应为 1000000
'claude-sonnet-4': { ..., context: 200000, ... }, // 应为 1000000

且 _resolveModelContextLimit() 完全依赖硬编码表，不查询 gateway 实测数据。

修复#

a) 修正 MODEL_CAPABILITIES 中的 Claude 4 系列 context 值

'claude-opus-4':   { ..., context: 1000000, ... },
'claude-sonnet-4': { ..., context: 1000000, ... },

b) _resolveModelContextLimit() 优先查 gateway 实测数据

function _resolveModelContextLimit(modelHint = '') {
  // 1. Gateway adapter-reported real data (single source of truth)
  try {
    const gw = require('../services/gateway/aiGateway');
    const instance = typeof gw.getInstance === 'function' ? gw.getInstance() : gw;
    if (instance?.getModelContextWindow) {
      const dynamic = instance.getModelContextWindow(lower);
      if (dynamic > 0) return dynamic;
    }
  } catch { /* gateway not ready yet */ }

  // 2. Fallback: MODEL_CAPABILITIES static table
  // 3. Unknown model → 128000
}

4. 上下文窗口全面去硬编码#

现象#

所有模型的 context window 都来自硬编码表（MODEL_CONTEXT_LIMITS / MODEL_CAPABILITIES），新增模型需要手动更新代码。

修复#

建立单一数据源架构：适配器实测 → gateway 缓存 → UI 显示。

a) 删除 hudRenderer.js 的 MODEL_CONTEXT_LIMITS 硬编码表

getContextLimit() 完全委托给 gateway：

function getContextLimit(model) {
  if (!model) return 0;
  try {
    const gw = require('../services/gateway/aiGateway');
    const instance = typeof gw.getInstance === 'function' ? gw.getInstance() : gw;
    if (instance?.getModelContextWindow) {
      return instance.getModelContextWindow(model);
    }
  } catch { /* gateway not ready */ }
  return 0;
}

5. CJK 孤字避免（wrapText 优化）#

现象#

行末刚好只剩一个 CJK 字符位置时，会出现孤字（如 "今" 独占行末，后续内容换行）。

修复#

在 wrapText 的 greedy wrap 中添加孤字检测：

const widthAfter = currentWidth + tok.cw;
const remaining = maxWidth - widthAfter;
// 如果当前 CJK 字符放入后，剩余空间不足以容纳下一个 CJK 字符
// 且下一个字符不是避头标点 → 提前换行，避免孤字
if (tok.cw === 2 && remaining >= 0 && remaining < 2 && ti + 1 < tokens.length) {
  const next = tokens[ti + 1];
  if (next.cw === 2 && !_NO_LINE_START.has(next.ch)) {
    lines.push(current + '\x1b[0m');
    current = ''; currentWidth = 0;
    continue;
  }
}
