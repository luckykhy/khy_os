一、改造摘要（已修致命缺陷 + 核心模块）#

本轮（执行臂为主，核心舱复核）共闭环 7 个致命缺陷：

#模块缺陷类别处置
1tools/executeCode.js代码执行 vm 逃逸（globalThis.constructor.constructor 经宿主全局对象逃逸）进程级真隔离：--permission 无能力子进程 + 三层防护，fail-closed，默认禁用兜底
2tools/createDocument.js任意写汇聚点（展开后绝对路径未复检）validateNoPathTraversal 封闭展开后 outputPath
3tools/pdfToWord.js任意写汇聚点封闭 outputPath（inputPath 读路径合法消费 /tmp，按 ext 受限，刻意不封）
4tools/imageOcr.js任意写汇聚点封闭 outputDocx（imagePath 读路径同上刻意不封）
5tools/imageGenerate.js最危险裸字节写汇聚点（无后缀约束 + 递归 mkdir）封闭展开后 target
6tools/inputValidators.js系统级提权残留：默认信任整个 home → ~/.ssh/authorized_keys、shell rc、autostart 全放行锚定 homedir 根的敏感 denylist（SSH/shell-rc/GPG/autostart/systemd/LaunchAgent），可信根内亦硬拦
7services/toolGuards.js editBoundaryGuard中央守卫层同病：第二条独立 home 提权路径防御纵深：接入 #6 同一 denylist 单一真源，trusted-root 放行前先拦

另：上一会话已闭环的 7 项执行臂修复（路径封闭「展开倒置」×5、SSRF 有符号 int32、executeCode 初版）记录于 memory/project_aerospace_refactor_engine.md。

核心舱复核：6 个关键文件（elf/syscall/vmm/ipc/pmm/process.c）确认沿用前轮加固——溢出安全 extent 检查、用户窗口封闭、wrap/内核半区拒绝、丢失唤醒死锁修复、Multiboot2 有界遍历、饱和 refcount。无新致命缺陷。

回归：工具层 167/167；守卫+工具+安全+验证器 241/241 跨 24 套件全绿，零回归。

二、遗留风险清单#

级别项说明处置
🟠executeCode 出站网络Node Permission Model 不闸断 outbound network爆炸半径有界（无能力子进程无 fs/密钥可达），需 OS 级出站沙箱才能完全闭环
🟡命令 denylist 混淆绕过securityGuardService 正则黑名单可经混淆（rm -r${IFS}-f 等）绕过模式过滤的固有局限，非离散缺陷；权威防线是审批层 + 防御纵深
⚪媒体工具读路径不封闭imageOcr.imagePath / pdfToWord.inputPath 刻意保留不封合法消费 /tmp 临时件，按 ext 受限，非裸密钥字节原语，低危
⚪imageDetect 读封闭已撤回循环 12 加读封闭致 os.tmpdir 夹具回归 7 例 → 立即撤回纯读工具无写汇聚点，撤回正确
⚙运维知情开关KHY_ALLOW_SENSITIVE_HOME_WRITE=1 / KHY_ENABLE_EXECUTE_CODE=1 重开对应向量设计内的知情决策，非缺陷
未扫显示台（TUI/GUI）按安全域律最低优先，仅核心稳定后才扫；3 连洁净已先触发停机渲染非执行，无致命类汇聚点，存活假定充分

三、系统现状评估（极端条件存活概率）#

  • 核心舱（内核）：6 关键文件全加固，溢出/越界/死锁/refcount 防线在位。极值风暴下 存活：高。
  • 执行臂（网关/工具执行器）：所有写汇聚点已封；任意写提权（含 home 持久化向量）在工具层 + 守卫层双闭环；代码执行隔离至无能力子进程；SSRF int32 闭环；资源枯竭有界（heap/buffer/ulimit/unref timer/出栈栈）。残留仅子进程出站网络 + 命令 denylist 混淆绕过，均有界/防御纵深。存活：由突击前的中 → 现高。
  • 显示台（TUI/GUI）：未扫（最低优先 + 提前触发停机）。渲染非执行，存活假定充分。

综合判定：核心舱与执行臂在极值风暴 / 资源枯竭并死锁 / 并发冲突三类极端推演下具备航天级存活能力；剩余残留为有界、需 OS 层能力或属知情运维决策，已逐条登记。引擎按「见好就收 / 抓大放小」原则于连续 3 次洁净扫描停机。
