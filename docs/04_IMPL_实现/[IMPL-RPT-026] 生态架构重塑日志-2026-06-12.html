生态架构重塑日志（Eco-Arch Log）#

目标：确立 khyos = 生态底座、khyquant = 首款应用，建立「应用接入标准」，
让 khyquant 成为符合标准的示范应用，实现底座与应用绝对解耦。
执行方式：无人值守一口气执行，保守防呆（不确定就保留+标注，绝不盲删导致崩溃）。

唯一契约文档：docs/生态架构/应用接入标准.md

一、五步执行结果#

步骤 1 — 剥离硬依赖 → 动态发现 ✅#

  • 全量扫描 khyos 核心源码（platform/khy_platform、platform/khy_os）对应用包的导入。
  • 结论：底座核心零应用硬依赖。khy_platform 中不存在任何 import khy_quant。
  • 唯一对 khy_quant_backend（底座自身 Node 后端的独立打包）的探测，已包在

try/except ImportError 内（platform/khy_platform/cli.py:62）——属底座定位自身后端，非依赖应用。

  • 底座经 importlib.metadata 的 khyos.apps entry_points 组 + ~/.khyos/apps/ 注册表目录

双路动态发现应用，有则加载、无则跳过、永不崩溃。

步骤 2 — 应用生命周期协议 ✅#

  • 新建单一真源 platform/khy_platform/app_protocol.py（零应用导入）：
    • 路径主权：base_home()→~/.khyos，app_home(name)→~/.<name>（含路径穿越防护），

STANDARD_SUBDIRS = (data, cache, models, logs)，ensure_home()。

  • 协议基类 KhyApp（抽象 standalone_init() / eco_init(ctx)）+ AppManifest / EcoContext。
  • 发现：discover_apps() 合并 entry_points（优先）与注册表，按 name 去重，任何异常不抛出。

步骤 3 — khyquant 双模自适应入口 ✅#

  • 新建 software/khyquant/khy_quant/app.py：KhyQuant(KhyApp) 示范实现，

对底座标准模块的导入做守卫——独立安装无底座也不崩溃。

  • khy_quant/cli.py：_detect_mode()（KHYOS_ECO_MODE 优先，否则探测底座 app_protocol 可导入，

默认 standalone）+ _ensure_app_home()，经 KHYQUANT_MODE / KHYQUANT_HOME 下传 Node 运行时。

  • pyproject.toml 声明 [project.entry-points."khyos.apps"] khyquant = "khy_quant.app:create_app"。

步骤 4 — 底座/应用 DB 物理隔离（路径所有权）✅（解析器层）/ ⚠️（活体迁移留待人工）#

  • services/backend/src/utils/dataHome.js 新增底座归属解析器 getBaseHome()→~/.khyos

（KHYOS_HOME 可覆盖）、getBaseDataDir()，与应用数据家 getDataHome() 物理分离。

  • 纯增量、不迁移活体数据：维持应用现有数据家，保证运行中的 khyquant 不被破坏。
  • 真正的「混用硬编码 ~/.khyquant 与解析器 ~/.khy 收敛」「底座/应用表物理拆分」

涉及活体数据迁移 → 见下方 [Eco-Arch-Unresolved]，按防呆规则不在无人值守流程内盲目执行。

步骤 5 — 模型/缓存/日志路径独立 ✅（标准层）#

  • 标准固化为 ~/.<app>/{data,cache,models,logs}，由 app_protocol.ensure_home() 统一创建。
  • 底座自身落 ~/.khyos/{data,cache,models,logs}。红线：互不读写对方目录/DB，跨域只走公共 API。
