## 怎么用这份文档（先读这段）

实施这种重塑**最忌讳**一句"照着文档实现"就甩给 AI——文档有 4 大块、触及十几个核心文件，一次性做必然失控、必然幻觉。

这份文档给你**两种用法**，按你的情况二选一：

- **🟢 一键自动（推荐给不想逐步盯的人）**：复制 [§0 的 goal 指令](#0-一键自动用-goal-完成推荐) 粘进 `/goal`，AI 自己按 B→A→C→D 跑完。**人工批准闸被换成"测试必须绿才放行"的自动闸**——AI 靠每阶段的数学性质测试自我验收，失败就停下报告，不会偷偷跳过。
- **🔵 逐步手动（想全程盯着）**：用 [§1 启动总纲](#1-启动总纲只发一次建立纪律) + [§2 阶段提示词](#2-阶段提示词逐阶段下发)，每阶段你亲自看一眼再放行。

无论哪种，三条铁纪律不变：**顺序锁死 B→A→C→D**（设计文档 §6 已论证第一刀必须是零回归的 §4.B）、**先读后改核对行号**、**失败就停不准粉饰**。

### 自动模式下"批准闸"换成了什么

| 手动模式靠 | 自动模式换成 |
|---|---|
| 你看完计划点头 | AI 先写测试、跑绿再继续——**测试就是裁判** |
| 你确认没跑偏 | 数学性质写成断言，跑红就卡住，AI 无法蒙混 |
| 你决定要不要进下一阶段 | 「当前阶段测试 + 回归套件全绿」才解锁下一阶段 |
| 你判断是否失败 | 撞到修不好的真实错误，AI 主动停下报告卡点 |

> 安全感来自哪：goal 模式不会让 AI"假装做完"——设计文档把完成标准定义成**真实命令/测试通过**，不是模型自述。这正是项目 `HARD_PROHIBITIONS`「禁止谎报未验证的成功」的兜底。

## §2 阶段提示词（批准计划后，逐阶段下发）

每阶段共享同一套执行步骤（下方「通用执行步骤」），只把**本阶段目标**与**必须断言的数学性质**替换进去。下面四段已经替你填好，直接复制对应阶段即可。

### 通用执行步骤（每阶段都附在末尾）

```text
执行步骤，逐条做、逐条汇报：
1. 读文档 §4.X 与 §6 对应行，读所有要改的现有文件，核对行号锚点。
2. 给出最小改动方案：复用了哪个现有零件、新增哪个"黏合剂"、为什么
   不需要碰其他文件。
3. 实施。阈值/权重一律 env 可调 + 常量默认（零硬编码）。
4. 写测试，必须断言本阶段的数学性质（见上）。
5. 跑测试 + 既有回归套件，贴真实输出。失败就停下报告，不准粉饰。

```

### 阶段 B —— 验收包做启发 h(s)（第一刀，零回归）

> 对应 §4.B。收益最大、风险最低：让验收标准从「终点裁判」变成「全程指南」，一举消灭三子系统不共享目标的一致性病灶。

```text
进入【阶段 B】= 设计文档 §4.B「用验收包做启发 h(s)」。受启动总纲铁律约束。

本阶段目标：
- 复用 buildAcceptancePack / acceptanceCriteria.js 已枚举的验收谓词 {φ_i}，
  定义启发 h(s) = Σ_{未满足 φ_i} ĉ(φ_i)，ĉ 取极保守下估（每项≥1 动作）。
- 把 h(s) 暴露给规划层与外层 harness：不再只在终点跑 delivery gate，
  而是全程可读 h(s)；当 h(s) 停滞或上升时触发一次只读 verify 校准信念。
- 这是纯增量，不改变现有决策的默认路径（feature 可灰度开关）。

必须断言的数学性质：
- 可采纳性：对任意状态，h(s) ≤ 真实到目标剩余代价（用构造测试验证下估）。
- 单调性：满足一项验收后 h 严格下降；无验收项时 h(s)=0 当且仅当 gate PASS。
- 零回归：现有 toolUseLoop / agenticHarnessService 测试套件全绿。

（粘贴「通用执行步骤」）
```

### 阶段 A —— 可逆性分层的 LRTA*（把盲重试变学习搜索）

> 对应 §4.A。核心洞察：Ralph 续轮环已是 LRTA* 外层试验环，只差价值回填。依赖阶段 B 的 h(s) 已就位。

```text
进入【阶段 A】= 设计文档 §4.A「可逆性分层 LRTA*」。受启动总纲铁律约束。
前置：阶段 B 的 h(s) 已合入。

本阶段目标：
- 在 agenticHarnessService.js:365 的 Ralph 续轮环里加入价值回填：
  每轮结束后 h(s) ← min_a [ g(s,a) + h(T(s,a)) ]，持久化到 boulderState SQLite。
  把「不学习的 restart」升级为「收敛的 learning restart」。不新建循环。
- 在 executeTool 入口按工具元数据(isReadOnly/isDestructive)做可逆性分层路由：
  · A_safe（只读/可逆）：允许有界前瞻/beam（宽度 k≤3，env 可调）锐化信念；
  · A_commit（不可逆）：禁止投机分支，一步最优承诺 + 高 Irrev 强制 plan-then-confirm
    （复用现有 requestPermission，不新建审批通道）。

必须断言的数学性质：
- 收敛/单调：跨 trial 价值回填使 h 单调不增（同实例第 k+1 轮 ≤ 第 k 轮）。
- 分层正确：只读动作才进入前瞻分支；任何不可逆动作绝不被投机执行（测试反例）。
- 预算守恒：前瞻只产生只读代价，IterationBudget 的不可逆步数不被前瞻消耗。

（粘贴「通用执行步骤」）
```

### 阶段 C —— 调度统一（UCB 路由 / 真均衡账号 / 抢占工作流 / 单断路器）

> 对应 §4.C。隔离在 gateway 与调度层，可灰度。其中 C-3 抢占工作流改动最大，放本阶段最后做。

```text
进入【阶段 C】= 设计文档 §4.C「调度统一」。受启动总纲铁律约束。
按子项顺序做，每个子项独立验收：

C-1 适配器路由：把 aiGateway.js:2115 的 score=basePriority×10+Σpenalty 换成 UCB1
    （臂回报=成功率×速度，cooldown 并入探索项）；保留 failoverOrderStore 作先验初值。
C-4 统一断路器：废弃 aiGateway 内联指数退避，全栈改用 circuitBreaker.js 三态机。
C-2 账号池：把 accountPool.js:3121 的粘性 MRU(ORDER BY last_used_at DESC)
    改成真负载均衡（true LRU 或 power-of-two-choices），兑现 'Balance' 名称。
C-3 工作流抢占：给 workflowRunWorker 一个 quantum（每 N 步经 boulderState 落盘即让出），
    引入 ready-queue + 抢占，避免长 run 饿死短 run。

必须断言的数学性质：
- C-1：UCB1 选择公式正确；长期 regret 随 N 次调用次线性增长（统计测试）。
- C-2：负载在账号间收敛均衡（非粘性热点）。
- C-3：run 到达 quantum 必让出；让出后可跨进程从检查点精确恢复。
- 全程零魔数：所有权重/阈值/quantum 走 env + 默认常量。

（粘贴「通用执行步骤」，每个子项做完单独贴测试输出再进下一子项）
```

### 阶段 D —— 约束格形式化（散落 if → 偏序）

> 对应 §4.D。触及 executeTool 与各 guard，需充分回归。

```text
进入【阶段 D】= 设计文档 §4.D「约束格」。受启动总纲铁律约束。

本阶段目标：
- 把六维 guards 的「硬拒/可审批/放行」形式化为偏序格：
  ⊥(红线: pathTraversal/SSRF/critical-destructive，恒不可行，审批不可松弛)
  ⊏ soft(editBoundary/priorRead/fileStale，guardApproval 是向上松弛算子，代价 λ_human)
  ⊏ ⊤(恒可行)。
- 给每个现有 guard 标注它在格中的位置，约束投影 Π_C 据此统一裁剪 A(s)。
- 保证 A(s) 永不为空：格顶恒含 ask_user / abort_with_reason（活性兜底）。

必须断言的数学性质：
- 活性：任意状态下 A(s) 非空（构造权限全拒场景，验证仍有 ask/abort 可选）。
- 红线不可松弛：⊥ 元经 guardApproval 仍被拒（测试反例）。
- 等价性：重构后对既有用例的拦截/放行结果与重构前完全一致（黄金回归）。

（粘贴「通用执行步骤」）
```

## §3 设计依据（这套提示词为什么这么写）

| 风险 | 提示词的对策 |
|---|---|
| **AI 幻觉行号/函数** | 铁律 2「先读后改 + 核对锚点漂移再动手」，把设计文档当真源而非记忆 |
| **一次性做炸** | 锁死 B→A→C→D 单阶段推进，每段要你批准 |
| **推倒重来** | 铁律 1 + §6 映射表把改动钉在「接黏合剂」上，禁止重写三大核心文件 |
| **数学性质失守** | 把可采纳性/单调性/收敛性/活性写成**测试断言**，不是口头承诺 |
| **假装成功** | 铁律 5 +「贴真实输出、失败就停」——即设计文档里 HARD_PROHIBITIONS 的精神 |
| **顺序贪心** | 启动总纲强制「先出计划、别写代码」，第一刀只能是零回归的阶段 B |
