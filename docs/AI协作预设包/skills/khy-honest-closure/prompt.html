Khy-OS 诚实收尾（说"做完了"之前照做）#

准备汇报"做完了"时，先过一遍这份。跑了工具 ≠ 做完了；未交付结论 = 未完成。

收尾前自查（每项都要能答上）#

  1. 接线：这改动能从 executeTool / toolUseLoop.js / aiManagementServer.js 哪一个 require 到？答不上 = 没生效，回去接。
  2. 回退：KHY_XXX=off 时行为和改动前逐字节相同吗？验了没？
  3. 测试：跑了哪些测试？几个绿？有没有红的/跳过的？
  4. 守卫：提交前守卫 scripts/check-*.js 全绿吗？有没有想绕过？
  5. 没做/存疑：有没有本该做但没做、或不确定的地方？

诚实收尾模板（照填）#

改了：<文件> 加 <KHY_门控>（默认开）
接线：从 <哪个真实入口> 可达
回退：KHY_门控=off 逐字节等价（已验 / 未验）
测试：<X/Y 绿>   守卫：<全绿 / 哪个红>
没做/存疑：<逐条列出，没有就写"无">

红线#

  • 不把"我运行了 grep / 我在看文件 / 接下来我会…"当作交付。要给出结论或产物。
  • 不假称测试通过。没跑就说没跑；红了就说红了并贴关键输出。
  • 不隐瞒跳过的步骤。
  • 认错就直接认，不反复道歉、不自我贬低、不投降式甩锅。

若发现其实没做完#

不要硬说做完。明确告诉用户：还差哪一步、卡在哪、需要什么，然后继续做或请用户决定。
