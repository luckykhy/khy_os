⚡ 一页速览（给你自己 · 先看这一篇）#

30 秒看懂：这是什么、这套包怎么用、守住哪几条、遇事去哪。 看完就能上手。

这套包是什么（一句话）#

以后你没有强 AI 了，只能用弱模型/陌生大模型，还只能靠 pip 发布。
这套包把「怎么跟 AI 讲清项目 / 让它别改坏 / 出错怎么修 / 下一步做什么」
写成现成文字，你复制粘贴给 AI 就行。你只需要会复制粘贴。

项目是什么（一句话）#

Khy-OS = 自研 x86_64 C 内核 + Node.js AI 后端（业务主体）+ pip 启动器，命令名 khy，靠 pip install khy-os 分发。

这套包怎么分（最重要，别搞混）#

目录给谁你何时进
🤖 给AI看/AI（整段复制发它）要发东西给 AI 时
🧑 给人看/你（操作与决策）你拿不准怎么办 时

标准三步（照做）#

  1. 讲清项目 → 开新 AI 会话，从 给AI看/项目情况说明-开场白.md 按对方模型选一档（A强/B普通/C小模型），整段发出去。
    • 想让 AI 一次就掌握全局？直接发 给AI看/总说明-一次读懂全局.md（自包含）。
  2. 立规矩 → 要它改代码前，把 给AI看/协作铁律.md 的「铁律速查卡」发它。
  3. 派活/排错 → 派活去 给AI看/任务派发卡.md 挑卡发；出错去 给AI看/错误自查手册.md 找条发。

遇事去哪（速查）#

你的处境打开

怕改坏/要发布/要回退给人看/命脉自保清单-给人.md
想完整了解怎么用给人看/使用说明-怎么用这套包.md

5 条你要守住的铁事实（AI 全忘了也别退）#

  1. pip 是命脉——改完要能打成 wheel 发布（手册 docs/07_OPS_运维/[OPS-MAN-042]）。
  2. 守卫红了不许提交——别让 AI 用 --no-verify 硬闯。
  3. 单文件不超 2500 行。
  4. 改动要能一键关掉回原样（默认开的 KHY_ 开关 + 逐字节回退）。
  5. 不确定先只读体检：node services/backend/bin/khy.js maintain freshness；慌了看 docs/传承/紧急恢复卡片.md。

盯 AI 的三个追问（它说"做完了"时问）#

  1. 「这改动接到 executeTool / toolUseLoop / aiManagementServer 哪个入口了？」答不上=没生效。
  2. 「把那个 KHY_ 开关设成 off，行为和改之前一样吗？」不一样=破坏了回退。
  3. 「守卫全绿吗？没绕过吧？」

要更细的操作看 使用说明-怎么用这套包.md；整套包入口 ../00_INDEX_总入口.md。
