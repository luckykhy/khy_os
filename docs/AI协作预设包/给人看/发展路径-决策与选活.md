# 🧭 发展路径 · 决策与选活（给你自己看）

> 你**想不出让 AI 做什么**时看这篇。它帮你决定「现在该做哪一档」，
> 选好后去 `给AI看/任务派发卡.md` 找对应的卡，连「派发头」一起发给 AI。

---

## 优先级心法（换任何 AI 都适用）

**先稳命脉，再补短板，最后才谈扩张。**
命脉 = pip 能发 + 守卫能拦 + 能一键回退。这三样任何时候都不能退化。

---

## 决策树：现在该做哪一档

```
项目现在稳吗？（守卫全绿？pip 能发？maintain freshness 无红？）
├─ 不稳 → 只做"稳命脉"（发《任务派发卡》里的"命脉自检"卡），别碰新功能。
└─ 稳   → 有具体痛点吗？
         ├─ 有 → 去"近期治理"挑最像的一张卡（T1~T5）。
         └─ 无 → 从"中期方向"挑一张（M1~M5）；想更远看"远期方向"（F1~F4）。
```

---

## 各档是什么、什么时候选（去任务卡发对应编号）

### 近期 · 稳命脉（不稳就只做这个）
- **命脉自检卡**：每个新 AI 接手先跑一遍，确认守卫/版本/上帝组件没恶化。
- 什么时候选：**每次换 AI 接手的第一件事**，或你感觉哪里不对时。

### 近期 · 治理候选（有痛点、想稳步推进时）
风险低、范围小、随时可做，是这个项目一贯的推进方式（每条=一个叶子+一个默认开开关）：

| 卡 | 做什么 | 什么时候选 |
| --- | --- | --- |
| **T1** | 给一个处理外部输入的地方补边界/安全硬化 | 想让项目更稳、更抗坏输入 |
| **T2** | 拆一个过大的文件（上帝组件） | 有文件超 2500 行、代码越来越难改 |
| **T3** | 适配一个新模型/provider | 想接入新模型、或某模型调不通 |
| **T4** | 补一个小模型兼容缺口 | 你常用的小模型跑工具循环出问题 |
| **T5** | 给某功能失败时补"诚实出口"（定性+脱敏+出路） | 某功能一失败就甩看不懂的原始报错 |

### 中期 · 能力补强（项目稳了再做，范围略大）

| 卡 | 方向 |
| --- | --- |
| **M1** | 联网搜索/网页读取更稳（多引擎/解码/超时） |
| **M2** | MCP 生态（对外暴露工具 / 一键装开源 server） |
| **M3** | 视觉/多模态路由完善 |
| **M4** | CLI/TUI 与 Claude Code 键位体验对齐 |
| **M5** | 内置模型开箱可用（装完就"已配置"） |

### 远期 · 方向（别让弱模型直接动手）
这些要**先写一份设计文档**再动手，弱模型容易搞砸，慎重：
- **F1** 内核侧能力深化（调度/IPC/文件系统/网络）
- **F2** 生态应用扩展（接更多应用到底座）
- **F3** 治理层收敛（消除重复实现，降复杂度）
- **F4** 跨平台构建与分发

## 怎么把一个方向变成给 AI 的活（三步）

```
1) 用上面的决策树选一档，记住卡编号（如 T2）。
2) 打开 ../给AI看/任务派发卡.md，复制"派发头" + 那张卡的整段。
3) 发给已经收到过"开场白+铁律速查卡"的 AI 会话。
   它会先告诉你要改哪个文件、加哪个开关、怎么回退——你确认后它才动手。
```

---

## 给你自己的三个判断原则

1. **拿不准就选低风险的**：T1/T5 风险最低，M/F 越往后越要谨慎。
2. **一次只派一件小事**：尤其对小模型，任务越小越不容易搞砸。
3. **稳压过快**：宁可这个月只安全地拆了一个大文件，也别让 AI 一次大改埋雷。
