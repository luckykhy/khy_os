## 🧩 skills/（装进 khy，指导弱模型现场执行）

文档是"讲清与传阅"，**skill 是"现场执行"**——装进 khy 后，模型在会话内随时 `/激活` 拿到本项目纪律。
你平时主要用小模型，把这套装上最省心。

| 入口 | 说明 |
| --- | --- |
| **[skills 集合总说明](skills/README.md)** | 怎么安装到 khy（`khy skill import` 或放 `~/.khy/skills/`）+ 8 个 skill 何时用哪个 |

一键装全部：
```bash
for d in docs/AI协作预设包/skills/*/; do node services/backend/bin/khy.js skill import "$d"; done
```

## ⏱️ 三步最短用法

1. **讲清项目**：开新 AI 会话，从 `给AI看/项目情况说明-开场白.md` 按对方模型选一档，整段发出去。
2. **立规矩**：要它改代码前，把 `给AI看/协作铁律.md` 的「铁律速查卡」发给它。
3. **派活/排错**：派活去 `给AI看/任务派发卡.md` 挑卡发；出错去 `给AI看/错误自查手册.md` 找条发。
   自己拿不准怎么选/怎么判断 → 看对应的 `给人看/` 文件。

## 5 个你要守住的铁事实（即使 AI 全忘了）

1. **pip 是命脉**：改完必须还能打成 wheel 发布（`docs/07_OPS_运维/[OPS-MAN-042] 发布手册`）。
2. **守卫是免疫系统**：守卫红了不许提交，别让 AI 用 `--no-verify`。
3. **单文件不超 2500 行**。
4. **改动要能一键关掉回原样**（默认开的 KHY_ 开关 + 逐字节回退）。
5. **不确定就先只读体检**：`node services/backend/bin/khy.js maintain freshness`；慌了看 `docs/传承/紧急恢复卡片.md`。
