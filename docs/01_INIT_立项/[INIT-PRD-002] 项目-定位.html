KHY OS — 项目定位#

状态: 草案 (2026-05-30) — 方向已确定，尚未最终敲定。

KHY 是什么？#

KHY OS 是一个 AI 原生操作系统项目。从自研 x86_64 混合微内核，到 AI 驱动的终端、WASM 应用运行时、多通道 AI 网关——作为一个垂直整合的平台统一起来。

与跑在现有操作系统之上的 AI 编程工具不同，KHY 将 AI 视为系统级能力——从内核的 IPC 和系统调用层到交互式终端，AI 都是原生的。

分层详解#

KHY Kernel（内核层）#

真实可启动的 x86_64 混合微内核，C 语言编写，汇编启动链。不是模拟器或封装——通过 GRUB 启动，管理物理/虚拟内存，运行协作式调度器，提供端口式 IPC。

核心能力：

  • 4 级 x86_64 分页，每进程独立地址空间
  • 端口式同步 IPC（32 端口，能力控制）
  • ELF 和 PE 双格式加载器（同时运行 Linux 和 Windows 可执行文件）
  • 内核级 VFS + RAMFS
  • 帧缓冲窗口管理器（16 窗口，Z 序排列）
  • MoonBit 语言集成（编译为 C，链接入内核）
  • 构建为可启动 GRUB ISO，可在 QEMU 中运行

KHY Shell（终端层）#

AI 原生终端，取代传统 shell 体验。这是当前主要的用户交互界面。

核心能力：

  • 100+ 命令，中文/英文/拼音别名
  • AI 编程助手，完整工具调用（文件操作、git、shell、搜索等）
  • 目标模式（Goal Mode）— 自主多步骤任务执行
  • 上下文管理，支持压缩与会话持久化
  • 继续回忆 — "继续" 自动重新注入原始任务上下文
  • 本地大脑服务 — 确定性操作零 token 消耗

KHY Gateway（网关层）#

多通道 AI 路由器，消除供应商锁定。

核心能力：

  • 16 个适配器：Claude、Codex、Kiro、Trae、Windsurf、Cursor、Ollama、本地 LLM、中继 API 等
  • 优先级级联 + 断路器 + 自动冷却
  • 协议转换（Anthropic / OpenAI / Gemini / Grok / Codex 格式）
  • 速率限制（Redis 支持）、OAuth 管理、密钥轮换
  • 跨所有连接通道的模型发现

KHY Apps（应用层）#

平台级应用运行时。

核心能力：

  • WASM 沙箱，M1 风格微内核 IPC 模拟
  • 三种 ABI 模式：numeric-v1、string-v2、json-v2
  • 基于能力的访问控制（IPC、NET、FS_READ、FS_WRITE、WINDOW、SHM）
  • 应用注册、生命周期管理、自动启动
  • 插件/扩展生态

khyquant（旗舰应用）#

AI 驱动的量化交易平台——KHY OS 上构建的第一个应用。展示平台处理真实工作负载的能力。

  • 多 Agent 分析：6 个分析师角色 + 多空研究员 + 交易执行者
  • 回测引擎：权益曲线、夏普比率、回撤分析
  • ML 推理（XGBoost/LightGBM）本地模型分析
  • A 股市场数据，多数据源（AKShare、TuShare、efinance、iFind）
  • 通达信公式解释器，兼容已有策略

安装#

pip install khy-os
khy --version

环境要求：Python >= 3.8，Node.js >= 18。支持：Linux / macOS / Windows。
