1. 如何运行#

../../node_modules/.bin/jest --rootDir . \
  tests/services/typeset tests/tools/renderDocument.test.js

端到端用例在检测不到 python-docx 时自动 skip（describe.skip），单测部分（防呆/解析/
模板/工具契约）无 python 依赖恒可跑。

3. 关键断言：核心诉求与防呆逐条对应#

诉求 / 防呆对应用例断言
①模型只管语义防呆: a LaTeX command … rejected、… an HTML tag … rejected夹带排版代码 → success:false，渲染前即拒
②模板驱动gbt7714 carries its 国标 signatures、overrides deep-merge …格式来自模板键值；覆盖深合并不重述基线
③原子化排版端到端 renders a real .docx …模型只给 Markdown+模板名，排版由程序落地

防呆③换页确定性[[newpage]] … pagebreak、horizontal rule is NOT a page break、blank lines never create a page break换页只来自哨兵/结构块；空行、--- 绝不换页

4. 端到端证据（python-docx 在位时）#

用例 renders a real .docx and verifies A4 + applied formatting（256ms）输入一篇含
标题/正文强调/列表/[[newpage]]/表格的 Markdown，模板 gbt7714，渲染后：

  1. 工具返回 success:true、template:"builtin:gbt7714"、validation.pageSizeA4:true、

validation.headingSizeOk:true。

  1. 再用独立 python-docx 探针反向解析产物 .docx：
    • abs(page_width - 210mm) < 1mm 且 abs(page_height - 297mm) < 1mm → A4 ✓
    • 段落文本含"引言"(H1) 与"方法"(H2) ✓
    • 遍历所有 run，统计漏设 w:eastAsia 的有文本 run 数 → 0 ✓
    • 输出 OK missing_eastAsia=0 sections=1

这证明写后校验+打补丁闭环真实生效（首次渲染时页脚页码域的 1 个 run 漏 eastAsia，被
_verify_and_patch 捕获并补丁——见 validation.patched）。
