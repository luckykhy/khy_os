# Windows UI Chat 回归检查清单

## 适用范围

本清单用于发版前验证 Windows 下 AI 对话页面兼容性，重点覆盖：

- 思考窗与流式输出行为
- 键盘输入安全（`ArrowUp` 历史泄漏 / IME 回车处理）
- 高 DPI 缩放与窄屏下的布局稳定性

建议测试环境：

- Windows 10/11
- Chrome、Edge
- 显示缩放：100%、125%、150%

## 前置条件

1. 前后端服务均已启动且可访问。
2. 可正常登录并进入 `AI 对话` 页面。
3. 至少有一个可用 AI 通道（流式或普通回退通道）。

## 测试用例

### TC-01 `ArrowUp` / `ArrowDown` 历史泄漏防护

步骤：

1. 打开 `AI 对话` 页面。
2. 保持输入框为空。
3. 连续按 `ArrowUp`。
4. 连续按 `ArrowDown`。

预期：

- 输入框保持为空。
- 不会回填历史聊天内容。
- 不会出现重复草稿或异常回显。

### TC-02 输入法回车安全（中文 IME）

步骤：

1. 切换中文输入法。
2. 在输入框中开始组词（候选框可见）。
3. 按 `Enter` 确认组词。
4. 组词结束后再次按 `Enter`。

预期：

- 组词阶段：仅上屏，不发送消息。
- 组词结束后：`Enter` 发送消息。
- `Shift+Enter` 插入换行。

### TC-03 思考窗流式输出

步骤：

1. 发送中等或较长问题。
2. 观察生成过程中的思考窗与回答区。

预期：

- 思考窗出现状态时间线（含状态更新/完成信息）。
- 助手回答区逐步增长（流式体验）。
- 最终状态为完成。

### TC-04 停止生成行为

步骤：

1. 发送较长问题。
2. 在输出仍进行中点击 `停止生成`。

预期：

- 输出快速停止。
- 思考窗记录“停止”动作。
- 手动停止后不会静默回退并继续生成。
- 若停止前尚无输出，助手消息显示 `已停止生成。`。

### TC-05 长文本换行与溢出

步骤：

1. 发送包含长路径、长 URL、长日志或无空格长串的文本。

预期：

- 聊天气泡正确换行。
- 页面无横向撑爆或布局错位。

### TC-06 Windows 缩放下头部与导航稳定性

步骤：

1. 在 125%、150% 显示缩放下打开页面。
2. 缩窄浏览器窗口（接近平板/移动宽度）。
3. 在总览页和聊天页之间切换。

预期：

- 头部操作区与菜单可正常使用。
- 品牌区、菜单区、操作按钮无重叠。
- 页面内边距与可读性保持正常。

## 通过标准

满足以下条件方可放行发版：

- 6/6 用例通过
- 0 个阻塞级问题
- 0 个输入安全回归
