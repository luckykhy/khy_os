项目痛点诊断报告#

  • 项目名称：Khy-OS / services/backend（khy-os-backend v0.1.98）
  • 审计时间：2026-06-13
  • 健康度评分：6 / 10（功能完整、测试覆盖充分、治理体系活跃；但承载严重结构性债务：超巨函数、跨层循环依赖、真实无界内存泄漏、网关热路径同步 I/O）
  • 技术栈：Node.js + Express 服务端；Sequelize / better-sqlite3 / pg / redis 数据层；ws WebSocket；ink + react 终端 UI（TUI）；winston 日志；prom-client + OpenTelemetry 可观测；jsonwebtoken / bcryptjs / helmet / express-rate-limit 安全栈。规模：src/ 下 916 个 JS 文件、约 300,316 LOC、436 个测试文件、43 个直接依赖。版本控制健康（50 commits，最近 2026-06-12）。

审计方法纪律：本报告所有结论均来自工具化客观证据（grep 静态模式扫描、require() 互边环检测、函数行距测量、调用点回溯），每个问题精确到 文件路径:行号，并附根因推演与可操作的 suggested_fix。无任何"凭直觉臆测"的笼统结论。
