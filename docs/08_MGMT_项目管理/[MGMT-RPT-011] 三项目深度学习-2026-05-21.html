二、30 项差距清单 (G1-G30)#

P0 — 核心体验 (影响每次使用)#

ID差距参考来源当前 KHY 状态改进方案
G1流式换行门控DeepSeek line_buffer.rs无，直接在 \n\n 切割新增 LineBuffer 类：push(delta) 追加，take() 只释放到最后一个 \n，thinking 绕过门控
G2自适应分块策略DeepSeek chunking.rs无，固定逐段 flush双档 Smooth/CatchUp + 迟滞控制器：queue≥160行 进入 CatchUp，<32行+250ms 冷却退出
G3Markdown 解析/渲染分离DeepSeek markdown_render.rsrenderMarkdownLite 单函数拆为 parseMarkdown(text) → AST + renderAST(ast, width) → lines，resize 只执行 render
G4主题扩充+色深检测DeepSeek palette.rs / OpenCode 33 主题2 主题，无色深检测新增 8+ 主题 (catppuccin/tokyo-night/nord/gruvbox/solarized/dracula/system)；检测 COLORTERM/WT_SESSION → TrueColor/256/16 三级降级
G5权限 diff 预览+自动学习OpenCode permission.tsx仅 yes/no 确认权限对话框显示 edit diff 预览；"Allow always" 写入运行时规则集；通配符模式匹配 src/**
G6会话原子写入+检查点DeepSeek session_manager.rs直接 writeFileSync改用 tmp+fsync+rename 原子写入；每次 AI 完整回复后 checkpoint；崩溃时从 checkpoint 恢复
G7统一 ErrorEnvelopeDeepSeek error_taxonomy.rs13 类分类但无统一信封新增 ErrorEnvelope { category, severity, recoverable, code, message }；所有错误源适配到 Envelope
G8首次引导流程DeepSeek OnboardingState无，直接进入 REPL状态机引导：Language → API Key → Trust Directory → Tips → Done；从 LANG 自动检测语言

P1 — 竞争力 (影响用户留存)#

ID差距参考来源当前 KHY 状态改进方案
G9子代理类型化角色DeepSeek 7 类型 / Hermes delegate通用 Worker + 深度控制定义 5 种角色 (General/Explore/Plan/Implementer/Verifier)，每种有独立 system prompt + 工具集白名单
G10并行工具安全分类Hermes _PARALLEL_SAFE_TOOLS无，工具串行执行标记 isConcurrencySafe → 只读工具并行 (read_file/grep/glob)；写入工具按路径分区并行；交互工具强制串行
G11目标持久化 (Ralph Loop)Hermes goals.py无持久目标系统/goal <text> 设置跨轮次目标；每轮 judge 判定是否完成；未完成自动 continuation；/subgoal 添加子目标
G12会话分支+全文搜索DeepSeek/OpenCode无分支，简单索引搜索/branch 从当前节点 fork 新会话；SQLite FTS5 全文搜索 + trigram CJK 支持
G13帧率限制器DeepSeek frame_rate_limiter.rs无，每次状态变化即刷新双档 120fps/30fps 限制器；低运动模式(reduce-motion)自动降频
G14记忆 Plugin 架构Hermes 8 个 memory provider无结构化记忆定义 MemoryProvider 接口；内置 MEMORY.md provider；支持外部插件 (mem0/retaindb)；FTS5 搜索
G15Kanban 任务板Hermes kanban_db.py文件型 TaskBoardSQLite WAL + CAS 原子 claim；7 状态 (triage/todo/ready/running/blocked/done/archived)；heartbeat TTL
G16Compaction 经济学DeepSeek 500K 硬地板简单 token 阈值压缩引入前缀缓存经济学：低 token 数不压缩(保护缓存)；默认 80% 窗口触发；工具结果优先裁剪
