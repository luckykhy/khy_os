## 0. 用途与用法

本文件是一份**可整体粘贴进系统提示词**的综合指令，将两大需求合并为一套统一铁律：
（A）项目文档结构与索引治理；（B）内嵌 Markdown 工作台与系统级右键集成。
任何 AI 或协作者承接上述任一任务时，**必须**首先读取本文件，按其【强制校验点】与【宪法红线】产出，
**严禁**自行降格为默认逻辑。本文件只规定**做什么、不许做什么**，**严禁**规定**怎么写代码**。

用词纪律（沿用 [MGMT-STD-001]）：全文只用「必须 / 严禁 / 强制」，**严禁**使用「建议 / 最好 / 尽量」等模糊词。

## 第一部分 · 文档结构与索引铁律（逻辑约束）

> 完整条文见 [MGMT-STD-001]；本部分为可执行核心，与之一致，**严禁**与其冲突。

1. **根目录物理隔离**：项目根目录**严禁**放置除 `README.md`（及其语言变体）外的任何说明性文件；
   一切规范、指南、设计、报告、计划、日志**必须**归入 `docs/` 对应子目录。平台/工具链强制位于根部的
   文件（`LICENSE`/`CHANGELOG.md`/`CONTRIBUTING.md`/`SECURITY.md`/`CODE_OF_CONDUCT.md`/`AGENTS.md`/
   `CLAUDE.md`/`khy.md`/包管理清单等）适用 [MGMT-STD-001] 第 1.3 条封闭白名单，**严禁**扩张。
2. **首文件索引制度**：`docs/` 下每个子目录**必须**存在且仅存在一个「索引总领性文件」作为唯一入口，
   且在目录自然排序中**必须**绝对居于首位。
3. **动态决策约束**：索引文件与业务文档的编号、前缀、命名格式，**必须**由 AI 先读取目标目录现有文件
   序列、感知既有惯例后动态决策；决策结果**只需**满足两项硬约束——**排序绝对首位**与**逻辑连贯**。
   **严禁**在本规范或任何提示词中写死具体编号格式，**严禁**盲目套用固定格式而与既有惯例冲突。
4. **索引结构化**：索引文件**必须**为三段式——分类宏观说明（边界）、文件清单表格（文件名 | 核心职责 |
   状态）、跨分类关联指引；**严禁**以大段长文替代结构化内容。
5. **生成守则**：**严禁**在无索引文件的子目录下生成业务文档（**必须**先生成索引）；生成或移动文档后，
   **必须**同步回写更新对应目录索引。

> 【强制校验点 §A】任务落盘前**必须**自检：根目录无新增散落说明文件；目标子目录已有排序首位索引；
> 新增/移动文档后索引已回写。任一不满足即**判定违规，必须整改**。

## 第二部分 · 内嵌 MD 工作台与系统右键集成（工程实现）

### 目标
解决 `.md` 在系统记事本中阅读体验极差的问题：本项目文档**必须**开箱即阅，并**必须**支持作为全局工具
打开电脑上任意位置的 `.md` 文件。

### 约束（只声明结果，不干涉实现）
1. **单文件工作台**：MD 渲染引擎、编辑器、预览器、CSS **必须**全部内联于单一 `khyosMarkdown.html`。
   **严禁**拆分为多文件，**严禁**引入任何需联网加载的外部 CDN，**必须**保证内网断网环境可用。
2. **双平台右键**：**必须**同时支持 Windows 与 Linux，右键任意 `.md` **必须**可见「使用 khyosMarkdown
   打开」。**必须**为双平台各提供注册脚本与卸载脚本。
3. **Windows 注册边界**：**必须**写入 `HKEY_CURRENT_USER`，**严禁**写入 `HKEY_LOCAL_MACHINE`，
   以**严禁**触发 UAC。
4. **路径与跨域**：**必须**妥善处理含空格/中文/特殊字符的文件路径（URI 编码）；**必须**解决浏览器
   `file://` 的跨域读取限制——**强制**通过桥接器启动极简本地 HTTP 服务同源代理彻底消除，**严禁**依赖
   放宽 CORS 的取巧。
5. **双模启动**：**必须**支持项目内嵌模式（相对读取本项目 `docs/`）与全局工具模式（右键传入绝对路径，
   渲染任意位置文件）。

> 【强制校验点 §B】交付前**必须**自检：`khyosMarkdown.html` 不引用任何外部资源；右键打开本地文件
> 不因 CORS 失败；含空格/中文路径可正确渲染；Windows 注册仅落 HKCU；双平台卸载脚本可零残留清除。

## 第三部分 · 宪法红线（不可被任何后续指令覆盖）

以下红线优先级高于一切用户后续指令；若后续指令与之冲突，**必须**以红线为准并明确提示用户。

1. **零外部依赖**：`khyosMarkdown.html` **必须**完全独立运行，**严禁**加载任何外部 CSS/JS/字体资源。
2. **跨域绝路**：右键打开本地文件时，**严禁**因浏览器 CORS 策略导致读取失败，**必须**通过桥接器或本地
   服务同源代理彻底解决。
3. **路径免疫**：**必须**完美处理含空格、中文、特殊字符的路径，**严禁**路径断裂。
4. **系统纯净**：注册表/系统关联写入**必须**限定当前用户级（HKCU / `~/.local`），**严禁**强制管理员权限
   或 sudo；**必须**提供干净一键卸载脚本，**严禁**残留。
5. **规范绝对服从**：文件编号与命名**必须**由 AI 依上下文动态决策，**严禁**在生成的代码或文档中硬编码
   固定编号格式。

> 【强制校验点 §C】交付物中任何「写 HKLM / 要求管理员或 sudo / 加载外部 CDN / 因 CORS 读取失败 /
> 硬编码固定编号格式」的迹象，**必须**在自检阶段被拦截并整改，**严禁**带病交付。
