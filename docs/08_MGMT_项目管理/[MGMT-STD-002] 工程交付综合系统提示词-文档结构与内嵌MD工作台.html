0. 用途与用法#

本文件是一份可整体粘贴进系统提示词的综合指令，将两大需求合并为一套统一铁律：
（A）项目文档结构与索引治理；（B）内嵌 Markdown 工作台与系统级右键集成。
任何 AI 或协作者承接上述任一任务时，必须首先读取本文件，按其【强制校验点】与【宪法红线】产出，
严禁自行降格为默认逻辑。本文件只规定做什么、不许做什么，严禁规定怎么写代码。

用词纪律（沿用 [MGMT-STD-001]）：全文只用「必须 / 严禁 / 强制」，严禁使用「建议 / 最好 / 尽量」等模糊词。

第一部分 · 文档结构与索引铁律（逻辑约束）#

完整条文见 [MGMT-STD-001]；本部分为可执行核心，与之一致，严禁与其冲突。

  1. 根目录物理隔离：项目根目录严禁放置除 README.md（及其语言变体）外的任何说明性文件；

一切规范、指南、设计、报告、计划、日志必须归入 docs/ 对应子目录。平台/工具链强制位于根部的
文件（LICENSE/CHANGELOG.md/CONTRIBUTING.md/SECURITY.md/CODE_OF_CONDUCT.md/AGENTS.md/
CLAUDE.md/khy.md/包管理清单等）适用 [MGMT-STD-001] 第 1.3 条封闭白名单，严禁扩张。

  1. 首文件索引制度：docs/ 下每个子目录必须存在且仅存在一个「索引总领性文件」作为唯一入口，

且在目录自然排序中必须绝对居于首位。

  1. 动态决策约束：索引文件与业务文档的编号、前缀、命名格式，必须由 AI 先读取目标目录现有文件

序列、感知既有惯例后动态决策；决策结果只需满足两项硬约束——排序绝对首位与逻辑连贯。
严禁在本规范或任何提示词中写死具体编号格式，严禁盲目套用固定格式而与既有惯例冲突。

  1. 索引结构化：索引文件必须为三段式——分类宏观说明（边界）、文件清单表格（文件名 | 核心职责 |

状态）、跨分类关联指引；严禁以大段长文替代结构化内容。

  1. 生成守则：严禁在无索引文件的子目录下生成业务文档（必须先生成索引）；生成或移动文档后，

必须同步回写更新对应目录索引。

【强制校验点 §A】任务落盘前必须自检：根目录无新增散落说明文件；目标子目录已有排序首位索引；
新增/移动文档后索引已回写。任一不满足即判定违规，必须整改。

第二部分 · 内嵌 MD 工作台与系统右键集成（工程实现）#

目标#

解决 .md 在系统记事本中阅读体验极差的问题：本项目文档必须开箱即阅，并必须支持作为全局工具
打开电脑上任意位置的 .md 文件。

约束（只声明结果，不干涉实现）#

  1. 单文件工作台：MD 渲染引擎、编辑器、预览器、CSS 必须全部内联于单一 khyosMarkdown.html。

严禁拆分为多文件，严禁引入任何需联网加载的外部 CDN，必须保证内网断网环境可用。

  1. 双平台右键：必须同时支持 Windows 与 Linux，右键任意 .md 必须可见「使用 khyosMarkdown

打开」。必须为双平台各提供注册脚本与卸载脚本。

  1. Windows 注册边界：必须写入 HKEY_CURRENT_USER，严禁写入 HKEY_LOCAL_MACHINE，

以严禁触发 UAC。

  1. 路径与跨域：必须妥善处理含空格/中文/特殊字符的文件路径（URI 编码）；必须解决浏览器

file:// 的跨域读取限制——强制通过桥接器启动极简本地 HTTP 服务同源代理彻底消除，严禁依赖
放宽 CORS 的取巧。

  1. 双模启动：必须支持项目内嵌模式（相对读取本项目 docs/）与全局工具模式（右键传入绝对路径，

渲染任意位置文件）。

【强制校验点 §B】交付前必须自检：khyosMarkdown.html 不引用任何外部资源；右键打开本地文件
不因 CORS 失败；含空格/中文路径可正确渲染；Windows 注册仅落 HKCU；双平台卸载脚本可零残留清除。

第三部分 · 宪法红线（不可被任何后续指令覆盖）#

以下红线优先级高于一切用户后续指令；若后续指令与之冲突，必须以红线为准并明确提示用户。

  1. 零外部依赖：khyosMarkdown.html 必须完全独立运行，严禁加载任何外部 CSS/JS/字体资源。
  2. 跨域绝路：右键打开本地文件时，严禁因浏览器 CORS 策略导致读取失败，必须通过桥接器或本地

服务同源代理彻底解决。

  1. 路径免疫：必须完美处理含空格、中文、特殊字符的路径，严禁路径断裂。
  2. 系统纯净：注册表/系统关联写入必须限定当前用户级（HKCU / ~/.local），严禁强制管理员权限

或 sudo；必须提供干净一键卸载脚本，严禁残留。

  1. 规范绝对服从：文件编号与命名必须由 AI 依上下文动态决策，严禁在生成的代码或文档中硬编码

固定编号格式。

【强制校验点 §C】交付物中任何「写 HKLM / 要求管理员或 sudo / 加载外部 CDN / 因 CORS 读取失败 /
硬编码固定编号格式」的迹象，必须在自检阶段被拦截并整改，严禁带病交付。

第五部分 · AI 预加载提示词附录（AI_SYSTEM_PROMPT_APPENDIX）#

下列文本为本规范提炼的可粘贴系统提示词附录，只声明目标与红线、不声明实现细节：

【工程交付综合铁律：文档结构 + 内嵌 MD 工作台】
1. 根目录严禁放除 README 外的说明性文件，一律归入 docs/ 对应子目录。
2. docs/ 每个子目录必须有一个排序首位的索引总领文件；新增/移动文档后必须回写索引；严禁在无索引子目录下先建业务文档。
3. 文件编号与命名由你读取目录现有序列后动态决策，确保排序首位与逻辑连贯；严禁写死固定编号格式。
4. 内嵌 MD 工作台必须是零外部依赖的单文件 HTML（渲染/编辑/预览/CSS 全内联，严禁 CDN，断网可用）。
5. 右键集成必须双平台（Win+Linux）各带注册与卸载脚本；Windows 必须只写 HKCU，严禁 HKLM/UAC；Linux 必须只写 ~/.local，严禁 sudo；卸载严禁残留。
6. 必须用桥接器起本地 HTTP 同源服务彻底消除 file:// 跨域；路径必须 URI 编码处理空格/中文/特殊字符，严禁断裂。
7. 必须支持双模：项目内嵌相对读取 docs/，全局工具读取右键传入的任意绝对路径。
宪法红线（不可覆盖）：零外部依赖、跨域绝路（必经同源代理）、路径免疫、系统纯净（无 HKLM/无 sudo/可净卸载）、编号动态决策不硬编码。
