二、断裂的 require 路径（9 处）#

2.1 ai-backend/src/services/toolCalling.js（6 处）#

ai-backend 的 toolCalling.js 从 backend 复制而来，大量引用 ../cli/、../models/、../services/resourceGuard 等 ai-backend 中不存在的模块。

行号原引用修复方式
~46require('../cli/handlers/data') — handleQuotetry/catch 降级，返回"仅在主 CLI 中可用"
~64require('../cli/handlers/backtest') — handleBacktestRun同上
~78require('../cli/handlers/data') — handleDataFetch同上
~92require('../cli/symbolResolver') — resolveSymbol同上
~108require('../services/resourceGuard') — withTimeout (execute_code)移除，改用 VM 内置 { timeout: 5000 }
~149require('../services/resourceGuard') — safeExec (shell_command)改用 child_process.execSync + 30s 超时

strategy_list handler（~206 行）：require('../models') 改为 require('@khy/shared/models')。

2.2 backend/src/services/gateway/aiGateway.js:539#

- require('@khy/shared/src/services/cacheService')
+ require('@khy/shared/services/cacheService')

路径多了一级 /src/，不匹配 @khy/shared 的 exports map。被 try/catch 包裹后静默失败，导致 Redis health store 功能永远无法启用。

2.3 backend/src/routes/marketData.js:164#

- const { importAllStocks, ... } = require('../../import-stock-data');
+ let importAllStocks, importSingleStock, STOCK_LIST;
+ try {
+   ({ importAllStocks, importSingleStock, STOCK_LIST } = require('../../import-stock-data'));
+ } catch {
+   return res.status(501).json({ success: false, message: 'import-stock-data 模块不可用...' });
+ }

backend/import-stock-data.js 文件不存在，调用 /api/market-data/import 时直接崩溃。改为 501 降级响应。

三、环境变量和配置默认值矛盾（5 处）#

3.1 DB_RETRY 代码 fallback 与 .env.example 不一致#

位置旧值新值
backend/server.js:1458`DB_RETRY_ATTEMPTS \\
backend/server.js:1460`DB_RETRY_DELAY \\

代码 fallback 与 .env.example 推荐值（20/8000）存在 2 倍差异。现已对齐。

3.2 Docker 密码默认值 changeme vs .env.example 的 change-me#

变量docker-compose.yml 旧值新值
REDIS_PASSWORD${REDIS_PASSWORD:-changeme}${REDIS_PASSWORD:-change-me}
POSTGRES_PASSWORD${POSTGRES_PASSWORD:-changeme}${POSTGRES_PASSWORD:-change-me}

不一致导致：如果用户从 .env.example 复制配置但未设置密码，Redis/PostgreSQL 容器内密码与应用端不匹配，认证失败。

3.3 TRADING_AGENT_STOCK_TIMEOUT_MS fallback 不一致#

位置旧值新值
backend/src/routes/tradingAgents.js:415'10000''15000'

packages/shared/src/config/env.js 中 applyEnvDefaults() 设为 15000ms，但路由中 fallback 写了 10000ms。现已统一为 15000ms。

3.4 .env.example 缺少 8 个环境变量声明#

补充了以下变量（注释形式，不影响默认行为）：

  • DEFAULT_ADMIN_PASSWORD / AI_MGMT_ADMIN_PASSWORD — 管理员密码覆盖
  • KHY_ADMIN_PASSWORD / KHY_EXPORT_PASSWORD — 系统级密码
  • CORS_ALLOWED_ORIGINS / AI_MGMT_CORS_ORIGINS — CORS 配置
  • DB_RETRY_ATTEMPTS / DB_RETRY_DELAY — 数据库重试参数
