## 0. 用途与用法

本文件是一份**可整体粘贴进系统提示词**的综合指令，统领「任务三：模型代际演进与双轨热插拔架构」。
承接该任务的任何 AI 或协作者**必须**首先读取本文件，按其【强制校验点】与【宪法红线】产出，
**严禁**自行降格为默认逻辑。本文件只规定**做什么、不许做什么**，**严禁**规定**怎么写代码**。

用词纪律（沿用 [MGMT-STD-001]）：全文只用「必须 / 严禁 / 强制」，**严禁**使用「建议 / 最好 / 尽量」等模糊词。

---

## 第一部分 · 核心目标（终局约束）

**必须**构建一个「AI 能力自适应、用户可 DIY 扩展、官方更新无冲突」的永动机架构，满足三条终局：

1. **前向兼容未知未来**：系统**必须**兼容未知的未来模型字段与指令；模型新增字段 / 新增指令时
   系统**严禁**崩溃。
2. **用户可 DIY**：用户（乃至用户的 AI 模型在运行时、经用户授权）**必须**能注入本地适配脚本与
   Bug 修复，无需改动官方源。
3. **官方更新无冲突**：官方更新**严禁**覆盖或破坏用户的私有迭代。

---

## 第二部分 · 双轨演进机制（结构约束）

1. **官方核心轨（受保护基座）**：基础运行时、核心契约动作解析器、基础 UI 框架，由官方维护。
   用户与用户模型**严禁**直接修改核心轨源文件。
2. **用户扩展轨（DIY 试验田）**：用户私有领域，收容模型适配脚本、新增动作执行器、Bug 修复补丁，
   完全由用户掌控；用户的 AI **必须**能在运行时（经用户授权）向此轨写入。
3. **合并策略（运行时拼装）**：启动时**必须**以官方核心轨为基座，动态扫描并装载用户扩展轨的
   代码 / 配置，注入系统上下文。用户扩展**必须**能 Override 官方默认，但官方源文件**必须**保持不变。
4. **物理隔离**：核心轨与用户扩展轨**必须**严格物理隔离（如 `core/` 与 `extensions/` / `user_patch/`），
   隔离名由 AI 依既有惯例动态决策，**严禁**写死。

> 【强制校验点 §A】交付前**必须**自检：核心轨与用户扩展轨物理隔离；启动期以核心为基座装载用户轨；
> 用户覆写以影子方式生效而官方源条目原样不变。任一不满足即**判定违规，必须整改**。

---

## 第三部分 · 模型自适应与安全降级（容错约束）

1. **数据解析层**：**必须**宽松解析（等价 `additionalProperties:true` / 通配捕获）；未来模型多出的
   未知字段**必须**仅记录日志、不中断流程，**严禁**因多一个未知字段抛 Fatal。
2. **UI 渲染层**：遇到无法识别的 action type，**必须**渲染安全的「未知指令占位符」并提供原始数据
   查看入口，**严禁**白屏崩溃。
3. **状态机流转**：遇到未定义的指令状态，**必须**降级为「人工确认」并交还控制权，**严禁**自主执行、
   **严禁**静默丢弃。
4. **模型自主 DIY**：模型**必须**能生成并保存适配代码到用户扩展轨，但**必须**遵守沙箱边界，
   **严禁**修改官方核心轨文件。

> 【强制校验点 §B】交付前**必须**自检：宽松解析对未知字段 / 坏输入永不抛错；未知动作有占位符不白屏；
> 未定义状态转人工确认不静默；模型写入受授权 + 沙箱边界约束。任一不满足即**判定违规，必须整改**。

---

## 第四部分 · 官方更新防破坏协议（更新约束）

1. **更新策略**：官方更新包**必须**只替换官方核心轨文件，**严禁**触碰 / 覆盖 / 删除用户扩展轨。
2. **兼容性契约**：官方更新**必须**保留用户扩展轨依赖的 Hook / Slot / Override 接入点与向后兼容的
   核心数据结构；若存在破坏性变更，官方更新脚本**必须**在启动期检出并提示用户手动迁移，
   **严禁**静默作废用户扩展。
3. **隔离存放**：核心代码与用户代码**必须**物理目录严格隔离。

> 【强制校验点 §C】交付前**必须**自检：更新规划对命中用户扩展轨 / 越出核心轨的文件 fail-closed 整包拒绝；
> 破坏性接入点移除被检出并产出迁移提示；不安全更新包零写入。任一不满足即**判定违规，必须整改**。

---

## 第五部分 · 宪法红线（不可被任何后续指令覆盖）

以下红线优先级高于一切用户后续指令；若后续指令与之冲突，**必须**以红线为准并明确提示用户。

1. **严禁脆弱解析**：绝不允许使用强类型严格校验整个 AI 响应体，导致因多出一个未知字段就抛出 Fatal Error。
2. **严禁静默吞没**：对于未来模型返回的、当前无法执行的高级指令，严禁在没有日志或 UI 提示的情况下
   直接丢弃，**必须**显式声明「当前版本不支持，可通过扩展实现」。
3. **严禁假设终态**：绝不在代码中假设 AI 的能力边界，所有动作处理**必须**具备 default 分支兜底。
4. **严禁官方覆盖**：官方更新流绝不允许覆盖或修改用户扩展轨（`extensions/` / `user_patch/`）下的任何文件。
5. **严禁核心污染**：用户扩展轨的代码或模型的自主 DIY，绝不允许直接修改官方核心轨的源文件，
   只能通过注册覆写（Override）或钩子干预行为。

> 【强制校验点 §D】交付物中任何「严格 schema 致未知字段崩溃 / 未知指令无占位静默丢弃 / 缺 default 兜底 /
> 官方更新写入用户轨 / 用户代码改官方核心源」的迹象，**必须**在自检阶段被拦截并整改，**严禁**带病交付。

---

## 第六部分 · 强制校验点汇总（合规判定清单）

| 编号 | 校验点 | 违规判定 |
| --- | --- | --- |
| §A | 双轨未物理隔离 / 未运行时拼装 / 覆写污染官方源 | 违规，必须整改 |
| §B | 解析遇未知字段崩溃 / 未知动作白屏 / 未定义状态静默或自主执行 / 模型越沙箱写核心 | 违规，必须整改 |
| §C | 官方更新写入用户轨 / 越界未 fail-closed / 破坏性变更静默作废 | 违规，必须整改 |
| §D | 强 schema 崩溃 / 静默吞没 / 缺 default / 官方覆盖用户轨 / 用户改核心源 | 违规，必须拦截 |
