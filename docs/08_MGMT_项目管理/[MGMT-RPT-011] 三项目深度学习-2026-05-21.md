## 二、30 项差距清单 (G1-G30)

### P0 — 核心体验 (影响每次使用)

| ID | 差距 | 参考来源 | 当前 KHY 状态 | 改进方案 |
|----|------|---------|-------------|---------|
| G1 | **流式换行门控** | DeepSeek `line_buffer.rs` | 无，直接在 `\n\n` 切割 | 新增 `LineBuffer` 类：`push(delta)` 追加，`take()` 只释放到最后一个 `\n`，thinking 绕过门控 |
| G2 | **自适应分块策略** | DeepSeek `chunking.rs` | 无，固定逐段 flush | 双档 Smooth/CatchUp + 迟滞控制器：queue≥160行 进入 CatchUp，<32行+250ms 冷却退出 |
| G3 | **Markdown 解析/渲染分离** | DeepSeek `markdown_render.rs` | `renderMarkdownLite` 单函数 | 拆为 `parseMarkdown(text) → AST` + `renderAST(ast, width) → lines`，resize 只执行 render |
| G4 | **主题扩充+色深检测** | DeepSeek `palette.rs` / OpenCode 33 主题 | 2 主题，无色深检测 | 新增 8+ 主题 (catppuccin/tokyo-night/nord/gruvbox/solarized/dracula/system)；检测 `COLORTERM`/`WT_SESSION` → TrueColor/256/16 三级降级 |
| G5 | **权限 diff 预览+自动学习** | OpenCode `permission.tsx` | 仅 yes/no 确认 | 权限对话框显示 edit diff 预览；"Allow always" 写入运行时规则集；通配符模式匹配 `src/**` |
| G6 | **会话原子写入+检查点** | DeepSeek `session_manager.rs` | 直接 `writeFileSync` | 改用 tmp+fsync+rename 原子写入；每次 AI 完整回复后 checkpoint；崩溃时从 checkpoint 恢复 |
| G7 | **统一 ErrorEnvelope** | DeepSeek `error_taxonomy.rs` | 13 类分类但无统一信封 | 新增 `ErrorEnvelope { category, severity, recoverable, code, message }`；所有错误源适配到 Envelope |
| G8 | **首次引导流程** | DeepSeek `OnboardingState` | 无，直接进入 REPL | 状态机引导：Language → API Key → Trust Directory → Tips → Done；从 `LANG` 自动检测语言 |

### P1 — 竞争力 (影响用户留存)

| ID | 差距 | 参考来源 | 当前 KHY 状态 | 改进方案 |
|----|------|---------|-------------|---------|
| G9 | **子代理类型化角色** | DeepSeek 7 类型 / Hermes delegate | 通用 Worker + 深度控制 | 定义 5 种角色 (General/Explore/Plan/Implementer/Verifier)，每种有独立 system prompt + 工具集白名单 |
| G10 | **并行工具安全分类** | Hermes `_PARALLEL_SAFE_TOOLS` | 无，工具串行执行 | 标记 `isConcurrencySafe` → 只读工具并行 (read_file/grep/glob)；写入工具按路径分区并行；交互工具强制串行 |
| G11 | **目标持久化 (Ralph Loop)** | Hermes `goals.py` | 无持久目标系统 | `/goal <text>` 设置跨轮次目标；每轮 judge 判定是否完成；未完成自动 continuation；`/subgoal` 添加子目标 |
| G12 | **会话分支+全文搜索** | DeepSeek/OpenCode | 无分支，简单索引搜索 | `/branch` 从当前节点 fork 新会话；SQLite FTS5 全文搜索 + trigram CJK 支持 |

| G15 | **Kanban 任务板** | Hermes `kanban_db.py` | 文件型 TaskBoard | SQLite WAL + CAS 原子 claim；7 状态 (triage/todo/ready/running/blocked/done/archived)；heartbeat TTL |
| G16 | **Compaction 经济学** | DeepSeek 500K 硬地板 | 简单 token 阈值压缩 | 引入前缀缓存经济学：低 token 数不压缩(保护缓存)；默认 80% 窗口触发；工具结果优先裁剪 |

## 三、设计模式学习清单

### 从 DeepSeek-TUI 学习

| 模式 | 实现位置 | KHY 应用场景 |
|------|---------|-------------|
| **迟滞控制器** | `chunking.rs` Smooth/CatchUp 不同进入/退出阈值 | 流式分块策略，避免状态边界振荡 |
| **两阶段渲染** | `markdown_render.rs` parse→AST, render(AST,width)→lines | markdown 渲染性能优化，resize 零解析成本 |
| **原子写入+检查点** | `session_manager.rs` NamedTempFile+fsync+persist | 会话持久化崩溃安全 |
| **Golden Master Testing** | snapshot/protocol/state 三项 parity 测试 | CI 质量门控，防止跨版本行为漂移 |
| **缓存经济学** | `compaction.rs` 500K 硬地板保护前缀缓存 | 避免低 token 时压缩导致缓存失效成本大于收益 |
| **版本计数器** | `history_version: u64` per-cell 版本追踪 | 增量渲染，只重绘变化的 cell |
| **三级色深适配** | `ColorDepth` TrueColor/256/16 + nearest_ansi16() | 兼容所有终端环境 |
| **Lease Pattern** | `RESIDENT_LEASES` 子代理文件所有权 | 防止多子代理同时修改同一文件 |

### 从 Hermes Agent 学习

| 模式 | 实现位置 | KHY 应用场景 |
|------|---------|-------------|
| **迭代预算** | `IterationBudget` 父90轮/子45轮 | 防止 agent 无限循环消耗 token |
| **并行安全分类** | `_PARALLEL_SAFE_TOOLS` + `_NEVER_PARALLEL_TOOLS` | 工具并行执行策略，只读并行/写入分区/交互串行 |
| **CAS 原子 Claim** | `kanban_db.py` BEGIN IMMEDIATE + status CAS | 多 worker 竞争任务的无锁分配 |
| **Plugin Provider ABC** | `MemoryProvider` 接口 + `discover_memory_providers()` | 可扩展记忆系统，支持第三方存储 |
| **Prompt 注入防护** | `_CONTEXT_THREAT_PATTERNS` regex 扫描 | 上下文文件安全加载 |
| **三态忙碌输入** | interrupt/queue/steer 模式 | (已实现) 继续优化 steer 注入时机 |
| **Goal-Judge 循环** | `goals.py` 目标 + 辅助模型判定 + continuation | 持久化跨轮次任务追踪 |
| **数据驱动皮肤** | `skin_engine.py` YAML 定义 colors+spinner+branding | 用户完全自定义视觉主题 |

### 从 OpenCode 学习

| 模式 | 实现位置 | KHY 应用场景 |
|------|---------|-------------|
| **Leader 键序列** | `@opentui/keymap` 超时序列 | 类 Vim leader 键扩展快捷键空间 |
| **命令面板** | `CommandPaletteProvider` Ctrl+P | 统一的命令/设置/会话发现入口 |
| **权限通配符匹配** | `evaluate.ts` last-match-wins + Wildcard | 精细的路径级权限控制 `src/**` → allow |
| **Schema 预编译** | `Schema.decodeUnknownEffect` 初始化时编译 | 工具参数校验一次编译多次复用 |
| **双进程 Worker** | `thread.ts` TUI 渲染 + Worker AI 后端 | 渲染不阻塞 AI 请求，AI 不阻塞 UI |
| **System 主题** | 从终端 16 色自动推导完整主题 | 自动适配用户终端配色 |
| **Toast 通知** | variant(error/warning/info) + 持续时间 | 非阻塞状态反馈 |
| **Doom Loop 检测** | 连续 3 次失败触发 | 防止 AI 陷入重复失败循环 |
